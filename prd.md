# PRD: KTC Claude Plugin Hub

## 1. 프로젝트 개요

- **서비스 이름**: KTC Claude Plugin Hub
- **한 줄 설명**: 클로드 코드 플러그인을 검색·설치·관리하고, 개발자가 직접 플러그인을 배포할 수 있는 통합 웹 플랫폼
- **핵심 가치**: 사용자 편의성 (쉽고 빠른 플러그인 탐색·설치) + 개발자 생태계 활성화 (누구나 플러그인을 만들고 배포)

---

## 2. 타겟 유저 및 사용자 시나리오

### 타겟 유저
- **일반 사용자**: 클로드 코드 플러그인을 검색·설치·관리하는 사용자 (플러그인 등록도 가능)
- **관리자(운영자)**: 플러그인 심사·승인, 공식 플러그인 배포, 플랫폼 운영

### 사용자 시나리오
1. **일반 사용자**는 클로드 코드의 기능을 확장하고 싶을 때 Plugin Hub에서 원하는 플러그인을 검색하고 설치한다
2. **일반 사용자**는 자신이 만든 플러그인을 등록하고, 다른 사용자들과 공유한다
3. **일반 사용자**는 설치한 플러그인을 대시보드에서 활성화/비활성화하며 관리한다
4. **관리자**는 공식 플러그인을 등록·배포하고, 사용자 등록 플러그인을 심사·승인한다
5. **관리자**는 부적절한 플러그인을 사후 비공개 처리한다

### 플러그인 노출 규칙
- 운영자가 배포한 공식 플러그인 → **상단 섹션**에 노출
- 일반 사용자가 등록한 커뮤니티 플러그인 → **하단 섹션**에 노출
- 자신이 올린 플러그인만 삭제 가능 (관리자는 모든 플러그인 삭제 가능)

## 3. 핵심 기능 명세

### 필수 기능 (MVP)
1. **플러그인 검색 및 필터링**: 키워드 검색, 카테고리별 필터링
2. **플러그인 상세 페이지**: 설명, 스크린샷, 다운로드 횟수, 평점/리뷰, 버전 정보
3. **플러그인 설치/삭제**: 원클릭 설치 및 삭제
4. **내 플러그인 대시보드**: 설치한 플러그인 활성화/비활성화, 설정 관리
5. **플러그인 등록/수정/삭제**: 일반 사용자 및 관리자 모두 플러그인 등록 가능
6. **버전 관리**: 플러그인 버전 업데이트 및 히스토리 관리
7. **관리자 심사/승인**: 사용자 등록 플러그인 승인 후 공개, 사후 비공개 처리
8. **공식 플러그인 배포**: 관리자가 등록한 플러그인은 즉시 공개 및 상단 노출
9. **평점 및 리뷰 시스템**: 로그인 사용자만 작성, 플러그인당 1인 1리뷰, 본인 플러그인 리뷰 불가
10. **카테고리 분류**: 플러그인을 카테고리별로 구분하여 탐색 용이
11. **다운로드 횟수 집계**: 플러그인별 다운로드 수 실시간 집계 및 표시

### 선택 기능 (추후 확장)
1. **소셜 로그인**: Google, GitHub 등 소셜 계정 연동
2. **플러그인 추천 시스템**: 사용 패턴 기반 개인화 추천
3. **알림 시스템**: 설치한 플러그인 업데이트 알림, 심사 결과 알림

## 5. 주요 화면 구성 및 User Flow

### 화면 목록
1. **메인 페이지**: 상단에 공식 플러그인 섹션, 하단에 커뮤니티 플러그인 섹션, 검색바
2. **플러그인 상세 페이지**: 설명, 스크린샷, 버전 정보, 리뷰/평점, 다운로드 수, 설치 버튼
3. **검색/탐색 페이지**: 카테고리 필터, 키워드 검색, 정렬 옵션 (인기순, 최신순, 평점순)
4. **내 플러그인 대시보드**: 설치한 플러그인 목록, 활성화/비활성화 토글, 설정
5. **개발자 콘솔**: 내가 등록한 플러그인 관리, 새 플러그인 등록, 버전 업데이트, 통계 확인
6. **관리자 페이지**: 심사 대기 플러그인 목록, 승인/반려, 공식 플러그인 관리, 비공개 처리
7. **로그인/회원가입 페이지**: 이메일/비밀번호 가입 및 로그인

### User Flow

```
[비로그인 사용자]
메인 페이지 → 플러그인 탐색/검색 → 상세 페이지 → 로그인 → 설치

[로그인 사용자]
메인 페이지 → 플러그인 검색 → 상세 페이지 → 설치 → 내 대시보드에서 관리

[플러그인 등록]
개발자 콘솔 → 새 플러그인 등록 → 관리자 심사 대기 → 승인 후 공개

[관리자]
관리자 페이지 → 심사 대기 목록 → 승인/반려 → 공식 플러그인 등록/관리
```

## 6. 상세 기능 명세 및 예외 처리

### 6.1 회원 관리
- 이메일/비밀번호 방식으로 회원가입 및 로그인
- 역할: 일반 사용자 / 관리자 (일반 사용자도 플러그인 등록 가능)
- 관리자 권한은 DB에서 직접 부여 (별도 관리자 가입 페이지 없음)
- **예외**: 이메일 중복 가입 시 → 에러 메시지 표시
- **예외**: 비밀번호 형식 미충족 시 → 조건 안내 메시지

### 6.2 플러그인 검색 및 탐색
- 키워드 검색 (플러그인명, 설명 대상)
- 카테고리별 필터링
- 정렬: 인기순(다운로드), 최신순, 평점순
- **예외**: 검색 결과 없음 → "검색 결과가 없습니다" 안내

### 6.3 플러그인 설치/삭제
- 로그인 사용자만 설치 가능
- 원클릭 설치 및 삭제
- 설치 시 다운로드 횟수 증가
- **예외**: 비공개 처리된 플러그인 설치 시도 → 안내 메시지

### 6.4 플러그인 등록/수정/삭제
- 일반 사용자: 등록 → 관리자 승인 후 공개
- 관리자: 등록 → 즉시 공개 (공식 플러그인, 상단 노출)
- 자신이 등록한 플러그인만 수정/삭제 가능
- 관리자는 모든 플러그인 수정/삭제/비공개 처리 가능
- **예외**: 중복 플러그인명 등록 시 → 에러 메시지
- **예외**: 삭제된 플러그인을 설치한 사용자에게 → 알림 제공

### 6.5 버전 관리
- 새 버전 업로드 시 버전 히스토리에 추가
- 사용자에게 업데이트 알림
- **예외**: 버전 번호 중복 시 → 에러 메시지

### 6.6 평점 및 리뷰
- 로그인 사용자만 리뷰 작성 가능
- 플러그인당 1인 1리뷰 (수정 가능)
- 본인이 등록한 플러그인에는 리뷰 불가
- 평점: 1~5점
- **예외**: 중복 리뷰 작성 시도 → "이미 리뷰를 작성했습니다" 안내

### 6.7 관리자 심사/승인
- 심사 대기 플러그인 목록 확인
- 승인 → 커뮤니티 섹션에 공개
- 반려 → 등록자에게 사유 안내
- 사후 비공개 처리 → 목록에서 숨김, 설치자에게 안내

## 8. 제약사항 및 보안 요구사항

### 제약사항
- 플러그인 파일 업로드 시 용량 제한 및 파일 형식 검증
- 관리자 권한은 DB에서 직접 부여 (관리자 가입 페이지 없음)

### 보안 요구사항
- JWT Access Token + Refresh Token 방식 인증
- 비밀번호 해싱 (bcrypt)
- 모든 API 요청 시 인증/인가 검증
- SQL Injection, XSS 등 기본 보안 대응
- CORS 설정으로 허용된 도메인만 API 접근
